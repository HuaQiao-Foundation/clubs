## Detailed Platform Profiles

### 1. Rotary Club Manager (Georgetown) ‚Äî 102/115 (89%)

**Cost (20/20):** $0 forever, CC BY 4.0 license, zero ongoing fees
**Mobile (19/20):** PWA (Progressive Web App), mobile-first design, 44px touch targets, offline-capable
**Global South (15/15):** Self-hosted fonts, no CDNs, Cloudflare Pages global edge, China-accessible
**Efficiency (15/15):** Purpose-built for Rotary, zero bloat, single-club focus
**Features (18/30):** ‚úÖ Speaker Mgmt (10/10 kanban), ‚ùå Attendance (0/5 - Phase 3), ‚úÖ Members (4/5), ‚úÖ Events (4/5), ‚ùå RI Integration (0/5)
**Open Source (15/15):** CC BY 4.0, community-owned, inspectable code, forkable

**Strengths:**
- ONLY open source Rotary platform globally
- Best-in-class speaker management (kanban workflow: Ideas ‚Üí Approached ‚Üí Agreed ‚Üí Scheduled ‚Üí Spoken ‚Üí Dropped)
- TRUE mobile-first (not retrofitted responsive)
- China-friendly architecture (self-hosted, no blocked dependencies)
- HuaQiao Foundation credibility (Frank Yih's Service Above Self Award)

**Weaknesses:**
- Missing attendance tracking (planned Phase 3)
- No RI Direct Connect (manual export required)
- Single-club only (no district-level features)
- No native iOS/Android apps (PWA only)

## Open Source as Competitive Differentiator

### Why Open Source Matters for Rotary Clubs

**1. Philosophical Alignment**
- **Rotary:** "Service Above Self"
- **Open Source:** "Knowledge sharing for the common good"
- **Perfect cultural fit** ‚Äî Not just software, but service to global Rotary community

**2. Data Sovereignty**
- **Clubs own their data** ‚Äî Not stored on ClubRunner's/DACdb's servers
- **Inspectable security** ‚Äî Code is auditable (critical for Global South privacy concerns)
- **No data harvesting** ‚Äî Proprietary platforms can change terms, sell data
- **Transparent governance** ‚Äî Community decides roadmap, not corporate board

**3. Sustainability Without Vendor Risk**
- **Vendor bankruptcy?** Community forks and continues
- **No forced upgrades** ‚Äî Clubs upgrade when ready
- **No price increases** ‚Äî Can't be held hostage by licensing changes
- **Community maintenance** ‚Äî Rotarian developers worldwide can contribute

**4. Global Accessibility at Scale**
- **Districts can self-host** ‚Äî One server, 50 clubs, $0 marginal cost
- **Zone-level collaboration** ‚Äî Asia-Pacific Rotary maintains regional fork
- **Community translations** ‚Äî Not waiting for vendor translation packages
- **Regional customization** ‚Äî Clubs in Malaysia fork differently than Kenya

**5. Innovation Speed**
- **No vendor roadmap delays** ‚Äî Best ideas from any club can be incorporated
- **Global developer community** ‚Äî Rotarian developers contribute features
- **Feature requests don't queue** ‚Äî Community prioritizes collectively

### License Strategy: CC BY 4.0 (Not MIT or GPL)

**Why Creative Commons Attribution 4.0?**

| Consideration | MIT/Apache | GPL | CC BY 4.0 | Georgetown Choice |
|---------------|-----------|-----|-----------|-------------------|
| **Attribution Required** | ‚ùå Optional | ‚úÖ Required | ‚úÖ Required | **Need credit** |
| **Copyleft (Share-Alike)** | ‚ùå No | ‚úÖ Yes | ‚ùå No | **Maximum freedom** |
| **Commercial Use Allowed** | ‚úÖ Yes | ‚úÖ Yes | ‚úÖ Yes | **Clubs can customize** |
| **Non-Technical Friendly** | ‚ö†Ô∏è Legal jargon | ‚ö†Ô∏è Complex | ‚úÖ Plain language | **Rotarians understand** |
| **Fork-Friendly** | ‚úÖ Yes | ‚ö†Ô∏è Must GPL | ‚úÖ Yes | **Regional forks OK** |

**Decision: CC BY 4.0**
- HuaQiao Foundation and Brandmine.io credited (attribution)
- Maximum freedom for clubs (no copyleft restrictions)
- Private customizations allowed (clubs keep modifications if desired)
- Plain language license (non-developers can understand)
