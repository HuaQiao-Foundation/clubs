# Handoff Prompts & Instructions

Detailed handoff instructions and troubleshooting guides for developers.

## Purpose

This directory contains:
- Handoff prompts for resuming work later
- Troubleshooting guides with investigation steps
- Instructions for other developers taking over tasks
- Context preservation for multi-session work

## Naming Convention

`topic-handoff.md` or `topic-instructions.md`

Examples:
- `typescript-error-handoff.md`
- `pwa-troubleshooting-instructions.md`

## Prompt Types

- **Handoff** - Instructions for resuming work in a future session
- **Troubleshooting** - Investigation steps and solution options for known issues
- **Instructions** - Step-by-step guides for specific implementation tasks
- **Context** - Background and decision rationale for complex features

## When to Create Prompts

Create a handoff prompt when:
- Work is blocked and needs to be resumed later
- A task needs to be delegated to another developer
- Complex troubleshooting requires documentation
- Implementation context needs preservation across sessions
